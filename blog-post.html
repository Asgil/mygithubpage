How to combine python and R to analyze data from Social Networks

Hello!

There are many so-called “Data Science wars” about which tool is better for big and fuzzy data analysis: Python or R. But most of them agree on a general idea that Python and R are very powerful both and it is mostly a question of choice. You can read more about pros and cons of Python and R here.

Personally I like to take the best from both of them, and in this post I want to give a quick tutorial on my approach of combining Python and R for the simplified ETL process.

We’ll follow the steps of the ETL process, adapting them for our needs:

    Step 1. Extract -> Get the data

    We will get some dummy data from the official Twitter API, for example some trending topics and we’ll collect tweets for that topic, which we will store in Json. Basically, there is no particular need to store the data for this exercise, we can extract and compute the needed metrics on the fly, but when you work with data mining, you know that the more you store - the better. To avoid situations when you lack some indicator in your data and you have to make more requests or even redo the crawl.

    Step 2. Transform -> Choose the data

    We will use Python to load json in memory, choose the metrics that’ll allow us to define the popularity of the topic and compute some additional metrics and export it in a format that’ll be convenient for the future analysis and plotting with R.

    Step 3. Load -> Plot the data

    Here we move a bit away from the classical meaning of the “Load” step of ETL process, we are not loading the data in the database, but we load data in R for the visualisation. We’ll do my favorite types of plots for the quantitative data analysis: CDF, CCDF and scatterplots with heatmaps.
